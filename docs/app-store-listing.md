## ✅ Submission-Checkliste

Wenn du Apple-Developer + Play-Console-Accounts hast:

**Vor dem Hochladen:**
- [ ] App-Icon/Splash in `mobile/app.json` gepflegt (Expo) — EAS generiert alle Größen automatisch beim Build
- [ ] Screenshots in allen Pflicht-Größen
- [ ] Privacy-Policy-URL erreichbar (✓ schon live: /datenschutz)
- [ ] AGB-URL erreichbar (✓ schon live: /agb)
- [ ] Support-Email aktiv (legal@chairmatch.de — braucht funktionierende Strato-Adresse!)
- [ ] In-App-Purchases korrekt deklariert (KEINE — Bezahlung via Stripe extern)
- [ ] Test-Account für Apple-Reviewer: tester@chairmatch.de / [generieren]

**Review-Zeiten (typisch):**
- Apple: 24-48 h
- Google: 2-7 Tage

**Häufigste Ablehnungsgründe:**
1. Login mit Demo-Account funktioniert nicht (Reviewer kann nichts testen)
2. Privacy-Policy-URL nicht erreichbar
3. Crash beim ersten Start
4. App nutzt In-App-Browser für externe Zahlung statt natives Payment (= Verstoß gegen Apple-Rules — wir nutzen Stripe via API, kein Browser-Embed)
